# maintainability

Check whether the change remains easy to understand, modify, and review.

## Checklist

- Responsibilities are placed in the correct layer/module.
- Naming is consistent with nearby code.
- Control flow is not unnecessarily complex.
- Reusable patterns are used instead of introducing one-off logic.
- Abstractions are not premature or overly broad.
- The diff is localized and reviewable.
- Comments explain non-obvious intent, not obvious code.
- The change follows existing project conventions.

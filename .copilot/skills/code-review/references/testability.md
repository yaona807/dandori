# testability

Check whether the change can be verified with focused tests and whether important cases are covered.

## Checklist

- New behavior has a focused test when appropriate.
- Regression-prone behavior is covered.
- Error paths are tested when meaningful.
- Boundary cases are tested when relevant.
- Tests are deterministic and avoid unnecessary broad integration scope.
- Test names describe behavior, not implementation details.
- Existing tests are updated only when behavior intentionally changed.
